## 1. 프로젝트 개요
- **프로젝트명**: React 컴포넌트 합성 및 기본 스타일링 실습 (`react_test_0714`)
- **한 줄 소개**: React의 핵심 구성 요소인 컴포넌트 모듈화, Props 연동, 그리고 Vanilla CSS를 활용한 레이아웃 스타일링 연습 웹앱
- **해결하려는 문제**: 단일 파일 구조에서 벗어나 컴포넌트를 독립적인 파일([App.jsx](file:///C:/Users/PC/Desktop/이희본/web/SPA/react_test_0714/src/App.jsx), [List.jsx](file:///C:/Users/PC/Desktop/이희본/web/SPA/react_test_0714/src/List.jsx), [Member.jsx](file:///C:/Users/PC/Desktop/이희본/web/SPA/react_test_0714/src/Member.jsx), [Picture.jsx](file:///C:/Users/PC/Desktop/이희본/web/SPA/react_test_0714/src/Picture.jsx))로 분리하고 이를 합성하여 화면을 구성하는 방법 학습.
- **목표 사용자**: React 컴포넌트 구조와 CSS 연동 기초를 공부하는 학습자 (이희본 님)

## 2. 핵심 목표
- **무엇을 만들고 싶은지**: 회원 목록(`Member`) 및 사진 목록(`List`)을 화면에 깔끔하게 보여주는 대시보드 형태의 React 컴포넌트 구조
- **왜 필요한지**: 실무에서 쓰이는 컴포넌트 재사용 패턴과 모듈식 파일 결합(import/export)을 손에 익히기 위함
- **성공 기준**: Vite 개발 서버 상에서 경고나 오류 없이 모든 컴포넌트가 조화롭게 렌더링되고, 개별 CSS 스타일이 온전히 적용되는 것

## 3. 사용자 시나리오
- **주요 사용자**: 웹페이지에 접속하여 회원 정보와 대표 사진 리스트를 확인하려는 방문자
- **사용자가 기대하는 행동**: 회원 카드와 이미지 리스트를 시각적으로 명확하게 구분하여 인지
- **주요 사용 흐름**:
  1. 웹 브라우저로 로컬 개발 서버 접속
  2. 상단 타이틀과 함께 회원 정보('문도박사')와 회원 사진이 렌더링된 것을 확인
  3. 목록에서 예쁜 반려동물 사진 카드가 섀도우 효과와 함께 표시되는 것을 확인

## 4. 기능 요구사항
- **필수 기능**:
  - **기능 1**: `Picture` 컴포넌트에서 이미지 태그(`img`)를 사용해 외부 펫 이미지(`pet-1.jpg`)를 성공적으로 호출 및 표시
  - **기능 2**: `Member` 컴포넌트에서 회원의 이름('문도박사')과 사진 카드를 조합하여 출력
  - **기능 3**: `List` 컴포넌트 내부에서 `Title`과 `Picture` 컴포넌트를 품고 있는 중첩(Nest) 컴포넌트 렌더링 구현
- **선택 기능**: 컴포넌트 단위로 Props를 전달하여 동적으로 이름이나 이미지를 변경할 수 있는 Props 인터페이스 구조 도입
- **예외 상황**: 이미지 URL 호출 실패 시 대체(alt) 텍스트나 기본 placeholder 이미지가 노출되도록 예외 처리

## 5. 화면/UX 요구사항
- **주요 화면**: 메인 단일 화면 (Single Page App)
- **화면 구성/레이아웃**: 
  - 회원 정보 영역은 세로 정렬(Flex Column) 구조
  - 타이틀, 이미지 카드, 회원 소개 블록이 카드 형태로 구획화됨
- **스타일/디자인 방향**:
  - 대표 타이틀: 눈에 띄는 `tomato` 컬러 텍스트 및 `lime` 배경색 적용
  - 회원 카드 배경: 차분한 블루 그레이(`rgb(98, 144, 175)`) 테두리와 글자 크기 조정
  - 이미지 카드: 둥근 모서리(`border-radius: 8px`)와 부드러운 하단 그림자(`box-shadow`) 스타일 부여
- **인터랙션 요구사항**: 향후 마우스 호버 시 카드 디자인이 살짝 떠오르는 모션 전환 효과 적용 고려

## 6. 데이터 및 연동
- **필요 데이터**: 회원의 기본 프로필 데이터 (이름, 이미지 소스 주소 등)
- **외부 API 또는 서비스**: 외부 이미지 호스팅 서버 (`http://qwerew.cafe24.com/images/pet-1.jpg`)
- **저장 방식**: 현재 단계에서는 별도의 DB 연동 없이 컴포넌트 내에 하드코딩된 Static / Mock 데이터 사용
- **인증/권한 요구사항**: 없음 (모든 사용자에게 공개)

## 7. 기술/제약 조건
- **사용 가능한 기술 스택**: React 19.2.7, Vite 8.1.1, Vanilla CSS
- **브라우저/디바이스 지원 범위**: 모던 웹 브라우저 (Chrome, Edge, Firefox 등) 최신 버전 지원
- **성능 요구사항**: 빠른 HMR (Hot Module Replacement) 지원으로 코드 변경 시 실시간 반영 보장
- **보안/접근성 요구사항**: `img` 태그에 웹 접근성을 위한 비어있지 않은 `alt` 속성 부여 및 `oxlint` 린터 경고 준수

## 8. 완료 기준
- **구현 완료 기준**: `App.jsx`에서 호출되는 모든 컴포넌트가 오류 없이 로드되고, 빌드가 정상 완료됨
- **테스트 기준**: `npm run lint` 수행 시 치명적인 에러(0 errors)가 나지 않고 빌드 완료 아티팩트가 생성됨
- **배포 조건**: 로컬 `dev` 서버 환경 실행 및 빌드 번들(dist) 파일 정상 생성 확인
